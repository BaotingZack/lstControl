# 桥式起重机三轴联动控制系统 — 方案设计文档

## 1. 系统概述

### 1.1 控制对象

三轴桥式起重机（类似港口集装箱起重机），轴定义：

| 轴 | 名称 | 运动方向 | 英文 |
|----|------|----------|------|
| X  | 大行车 | 沿跑道/导轨 | Bridge |
| Y  | 小行车 | 沿桥架横向 | Trolley |
| Z  | 抓钩 | 升降 (Z=0地面, Z>0上方) | Hoist |

X-Y 构成水平面，Z 独立升降。三轴在物理上可同时运动，但抓钩带货平移时必须在安全高度之上。

### 1.2 控制需求

- **输入**: 起始位置 (取货点) `(sx, sy, sz)`、终止位置 (卸货点) `(ex, ey, ez)`
- **状态**: 三轴实时位置 + 速度
- **联动**: 三轴可同时运动，多轴联动时 XY 同步到达（直线轨迹）
- **速度限制**: 最大速度可配，默认 XY=0.3 m/s, Z=0.2 m/s
- **零速死区**: 到位后速度 < 0.01 m/s 强制归零（模拟伺服 zero-speed 窗口）

## 2. 运动序列 (8 阶段)

```
初始状态: 起重机位于 (x0, y0, z0)

Phase 1 (APPROACH):    X→sx, Y→sy, Z→safe_z   三轴同时联动
                       Z 先到 safe_z 则等待，XY 都到 → Phase 2

Phase 2 (DESCEND):     Z→sz                    抓钩降到取货高度

Phase 3 (GRAB):        延时 0.5s                模拟夹具抓取

Phase 4 (LIFT_CARGO):  Z→safe_z                带货上升到安全高度

Phase 5 (TRANSFER):    X→ex, Y→ey              XY 联动平移 (Z保持safe_z)

Phase 6 (LOWER_CARGO): Z→ez                    抓钩降到卸货高度

Phase 7 (RELEASE):     延时 0.5s                模拟释放

Phase 8 (RETURN):      Z→z0                    空钩升回初始高度

safe_z = max(sz, ez) + safe_height_offset (默认 1.0m)
```

**关键约束**:
- Phase 1: Z 到 safe_z 后**等待** XY，XY 都到才触发 Phase 2
- Phase 4: **带货上升**后再平移，防止碰撞
- Phase 5: XY 时间同步，轨迹为直线（非 L 形折线）
- **Z-only 阶段** (P2/P4/P6/P8): XY 必须保持静止 (位置保持)

## 3. 控制架构 — S曲线 + 级联速度控制

### 3.1 整体架构

```
                  ┌─────────────────┐
                  │   S曲线轨迹规划   │  jerk-limited 7段式剖面
                  │  (p_ref, v_ref)  │
                  └────────┬────────┘
                           │
              ┌────────────┤
              ▼            ▼
         p_ref         v_ref (前馈, 主导运动)
              │            │
              ▼            │
    ┌─────────────────┐    │
    │  位置误差 e_p    │    │
    │  = p_ref - p    │    │
    └────────┬────────┘    │
             │             │
             ▼             ▼
    ┌──────────────────────────┐
    │  PLC 速度指令:            │
    │  v_cmd = v_ref           │   ← 前馈主导
    │        + Kp·e_p          │   ← 位置修正 (≤15% v_max)
    │  限幅 ±v_max             │
    └──────────┬───────────────┘
               │
               ▼
    ┌──────────────────┐
    │  伺服驱动:        │
    │  velocity = v_cmd │   速度模式, 瞬时响应
    └──────────┬───────┘
               │
               ▼
    ┌──────────────────┐
    │  机械:            │
    │  p += v · dt      │   欧拉积分
    └──────────────────┘
```

### 3.2 S曲线轨迹规划 (`s_curve.py`)

**为什么用 S曲线**: 工业伺服标准做法，限制 jerk（加加速度），产生平滑启停，减少机械冲击和吊钩摆动。

**7段式 jerk-limited 剖面**:

```
  v ^
    |        _______
    |       /       \
    |      /         \      ← 平滑加速/减速, 无突变
    |     /           \
  0 |____/_____________\____> t
    |T1|T2|T3| T4 |T5|T6|T7|

  T1: 加加速 (jerk>0, a 线性增大)
  T2: 匀加速 (jerk=0, a 恒定)
  T3: 减加速 (jerk<0, a 线性减小到0)
  T4: 匀速   (jerk=0, a=0)
  T5: 加减速 (jerk<0)
  T6: 匀减速 (jerk=0)
  T7: 减减速 (jerk>0, a 线性减小到0)
```

**自动判断逻辑**:
- 长距离: 完整 7 段 (可达 v_max 和 a_max)
- 中距离: 无 T4 匀速段 (可到 a_max, 到不了 v_max)
- 短距离: 纯 jerk 加减速 (到不了 a_max)
- 超短距离 (<0.7m): 缩减 Tj 匹配位移

**关键参数**:

| 参数 | XY 默认值 | Z 默认值 | 说明 |
|------|-----------|----------|------|
| max_velocity | 0.3 m/s | 0.2 m/s | 最大速度 |
| max_acceleration | 0.2 m/s² | 0.15 m/s² | 最大加速度 |
| max_jerk | 0.15 m/s³ | 0.1 m/s³ | 最大加加速度 |

### 3.3 速度控制器 (`pd_controller.py`)

**设计理念**: 模拟 PLC 编程模式。PLC 输出**速度指令**给伺服驱动，驱动内部闭环调速。PLC 不做加速度环。

```python
# 核心公式 (VelocityController.update)
e_p = p_ref - p_actual                # 位置误差 [m]
delta_v = Kp * e_p                    # 速度修正量 [m/s]
delta_v = clamp(delta_v, ±v_max*15%)  # 限制修正量
v_cmd = v_ref + delta_v               # 合成速度指令
v_cmd = clamp(v_cmd, ±v_max)          # 限幅
return v_cmd
```

**为什么限制位置修正为 v_max 的 15%**:
- S曲线 `v_ref` **主导运动**（提供平滑加/减速）
- 位置修正只是**微调**（补偿跟踪误差）
- 如果修正过大，会在 S曲线减速段产生冲突：S曲线要减速，位置环要加速追赶 → 过冲震荡

### 3.4 XY 时间同步

**问题**: X 走 6m，Y 走 3m，都用 0.3m/s → Y 先到等 X 10s → XY 轨迹呈 **L 形折线**。

**方案**: 以最慢轴时间为基准 `T_max`，二分搜索缩减快轴 `v_max`，使两者同时到达。

```
T_x = S曲线估算时间(6m, 0.3m/s) ≈ 22.8s
T_y = S曲线估算时间(3m, 0.3m/s) ≈ 12.8s
T_max = 22.8s → v_y_sync ≈ 0.146 m/s (二分搜索)
```

**效果**: XY 从 L 形折线变为**平滑对角线**。

### 3.5 零速死区 + 锁定

**工程背景**: Siemens SINAMICS `p1226` / Yaskawa `Pn507` — 伺服驱动的 zero-speed 窗口。

```
if |位置误差| < 到位容差(0.01m) AND |速度| < 零速阈值(0.01m/s):
    velocity = 0           # 强制归零
    locked = True          # 锁定 (不再响应指令, 模拟抱闸)
```

**锁定后**: 跳过速度指令更新，轴保持零速。换阶段时解锁。这消除了到位后 ∼0.005 m/s 的微爬行。

## 5. 方案对比分析

### 5.1 三种控制架构对比

| | **S曲线+级联PD(加速度)** | **S曲线+速度P(当前)** | **距离比例+斜坡** |
|----|------------------------|---------------------|-------------------|
| **PLC 输出** | 加速度指令 | **速度指令** | 速度指令 |
| **轨迹来源** | S曲线(p,v,a前馈) | S曲线(p,v前馈) | 无轨迹, 纯 P 反馈 |
| **速度环** | 内部 PD (kp_vel, kd_vel) | 无 (驱动闭环) | 驱动 ramp |
| **参数数量** | 7+ | **5** (精简) | 4 |
| **速度曲线** | S形 (jerk-limited) | S形 (jerk-limited) | 梯形 (exponential) |
| **速度平滑度** | ★★★★★ | ★★★★★ | ★★★ |
| **PLC 实现难度** | 复杂 (需加速度环) | **简单** (单行公式) | 最简单 |
| **抗扰动** | 强 (双环) | 中 (单环+前馈) | 弱 (纯反馈) |
| **实际贴合度** | 伺服内部调试 | **PLC+驱动标准做法** | 简化近似 |

**选择 S曲线+速度P 的理由**:
- PLC 只输出速度指令 → 符合实际 PLC 编程习惯
- S曲线提供平滑前馈 → 速度曲线高质量
- 参数精简 (kp_pos + v_max + a_max + j_max → 5个) → 易调参
- 位置修正限幅 → 防止过冲震荡

### 5.2 关键 Bug 修复记录

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | 速度超标 (5.46 m/s) | PD v_cmd 无限幅 | 加 ±v_max 限幅 |
| 2 | Z 速度正负交替 | S曲线"达不到a_max"分支遗漏 Tv 计算 | 补全匀速段时间 |
| 3 | P5 Vx 从-0.1跳到-0.25 | S曲线 `_compute_accel_end_pos` 重复加 `_p0` | 移除重复偏移 |
| 4 | Z-only 阶段 XY 飞 0.81m | S曲线最小位移 ~0.71m, D=0.002m 仍规划完整剖面 | 超短距离缩减 Tj |
| 5 | XY 轨迹 L 形折线 | 各轴独立速度, 快轴先到等慢轴 | XY 时间同步 |
| 6 | Phase 标签 P1 缺失 | `phase_boundaries[0][1]` 取到 P2 而非 P1 | 改用 `history[0]['phase']` |
| 7 | 到位后微爬行 (0.005m/s) | PD 持续输出微小修正 | 零速死区 + 锁定 |

### 5.3 参数调优指南

| 参数 | 默认值 | 作用 | 调大效果 | 调小效果 |
|------|--------|------|----------|----------|
| `kp_pos` | 0.5 | 位置修正强度 | 跟踪更紧, 但减速段可能震荡 | 更平滑, 但跟踪误差大 |
| `max_velocity_xy` | 0.3 | 大小行车限速 | 更快, 但 jerk 更大 | 更慢更平稳 |
| `max_acceleration_xy` | 0.2 | S曲线加速度 | 启停更快 | 更平滑 |
| `max_jerk_xy` | 0.15 | S曲线 jerk | 加速更猛 | 过渡更缓 |
| `velocity_deadband` | 0.01 | 零速阈值 | 早归零, 可能有静差 | 更精确但可能爬行 |
| `v_correction_max` | v_max×15% | 位置修正上限 | 跟踪更好 | 减速更稳 |
